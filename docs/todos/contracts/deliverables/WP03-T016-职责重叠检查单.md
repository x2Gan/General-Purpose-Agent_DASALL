## 2. 8 对象 × 5 域互斥矩阵

下表列出每个主链路对象在 5 大跨域方向上的禁止字段类别。"✗" 表示该对象必须禁止该域的字段入侵。

| 主链路对象 | Prompt 域 | Recovery 域 | Worker 域 | Runtime 域 | Provider 域 |
|---|---|---|---|---|---|
| **AgentRequest** | ✗ rendered_prompt | ✗ retry_count, backoff_ms | ✗ worker_task_id, multi_agent_mode | ✗ current_step, fsm_state | ✗ model_selection, vendor_tool_schema |
| **GoalContract** | ✗ rendered_prompt, model_provider_args | ✗ retry_counters, circuit_state, checkpoint_ref | ✗ worker_assignment | ✗ fsm_state, plan_graph, step_list | ✗ vendor_tool_schema |
| **ContextPacket** | ✗ final_messages, rendered_prompt, provider_payload | ✗ working_memory_snapshot, pending_action | ✗ worker_results | ✗ payload, error, side_effects, success | ✗ provider_payload |
| **Observation** | ✗ summary, key_facts, confidence (→Digest) | ✗ fsm_state, retry_counters | ✗ worker_lease | ✗ belief_patch, plan_patch | ✗ rendered_prompt |
| **ObservationDigest** | ✗ rendered_prompt | ✗ fsm_state, retry_counters | ✗ worker_lease | ✗ payload, success, error, side_effects, tool_call_id, duration_ms | ✗ provider_payload |
| **BeliefState** | ✗ final_messages, rendered_prompt, prompt_bundle | ✗ working_memory_snapshot, pending_action | ✗ worker_results | ✗ payload, error, success, side_effects, fsm_state | ✗ provider_payload |
| **Checkpoint** | ✗ rendered_prompt | ✗ failure_root_cause, belief_patch (→Reflection) | ✗ worker_results, merged_result | ✗ full BeliefState embedding | ✗ provider_payload |
| **AgentResult** | ✗ rendered_prompt | ✗ internal_checkpoint, fsm_state | ✗ worker_results, conflicts, merged_result | ✗ observation_history, plan_history | ✗ provider_payload |

**检查结论**：8 个对象在 5 个域方向上均已定义禁止字段规则，无遗漏行。

## 3. 相邻对象字段对称互斥表

T015 流图定义了 4 组关键相邻对象对，其字段必须互斥（任一字段不得同时为两个相邻对象的必填字段）。

### 3.1 Observation ↔ ObservationDigest（执行记录 vs 推理投影）

| Observation 专属字段 | ObservationDigest 专属字段 | 互斥判定 |
|---|---|---|
| payload（原始执行结果，R） | summary（压缩摘要，R） | ✓ 互斥 |
| success（布尔结果，R） | confidence（置信度，R [0.0,1.0]） | ✓ 互斥 |
| error（错误详情，O） | key_facts（关键事实，R） | ✓ 互斥 |
| side_effects（副作用，O） | citations（引用来源，R） | ✓ 互斥 |
| tool_call_id（工具调用 ID，O） | omitted_details（省略细节，O） | ✓ 互斥 |
| duration_ms（耗时，O） | — | ✓ 无重叠 |

**共享字段**：observation_id（关联用，非职责重叠）。  
**结论**：✅ 完全互斥，无重叠。

### 3.2 BeliefState ↔ Checkpoint（认知状态 vs 恢复快照）

| BeliefState 专属字段 | Checkpoint 专属字段 | 互斥判定 |
|---|---|---|
| confirmed_facts, hypotheses, assumptions（认知三元组，R） | state（FSM 枚举，R） | ✓ 互斥 |
| evidence_refs（证据引用，R） | step_id（步骤位置，R） | ✓ 互斥 |
| confidence [0.0,1.0]（认知置信度，R） | working_memory_snapshot（内存快照引用，R） | ✓ 互斥 |
| active_plan_summary, unresolved_questions, reasoning_trace（O） | pending_action, retry_count, budget_remaining, parent_checkpoint_id（O） | ✓ 互斥 |

**共享字段**：request_id（关联追踪用）。Checkpoint 用 belief_state_ref（string）而非嵌入完整 BeliefState。  
**结论**：✅ 完全互斥，无重叠。

### 3.3 GoalContract ↔ ContextPacket（目标锁定 vs 语义上下文）

| GoalContract 专属字段 | ContextPacket 专属字段 | 互斥判定 |
|---|---|---|
| goal_id, goal_description, success_criteria（R） | user_turn, recent_history（R） | ✓ 互斥 |
| constraints, status, approval_policy（R/O） | memory_facts, evidence_snippets, tool_visibility（O） | ✓ 互斥 |
| budget_override（O） | budget_report（O） | ✓ 语义不同（override 是约束，report 是状态） |

**关联字段**：request_id（追踪用）；goal_description → current_goal_summary（单向压缩，非重叠）。  
**结论**：✅ 完全互斥，无重叠。

### 3.4 AgentRequest ↔ AgentResult（入口 vs 出口）

| AgentRequest 专属字段 | AgentResult 专属字段 | 互斥判定 |
|---|---|---|
| user_input, request_channel, session_id（R） | response_text, structured_payload, task_completed（R） | ✓ 互斥 |
| trace_id, priority, tags（R/O） | result_code, status, checkpoint_ref（R/O） | ✓ 互斥 |
| runtime_budget, context_hint（O） | error, citations, traces（O） | ✓ 互斥 |

**共享字段**：request_id（关联追踪用）。  
**结论**：✅ 完全互斥，无重叠。

## 4. 域归属唯一性规则

每个主链路对象归属唯一的架构域，不得跨域持有必填字段：

| 主链路对象 | 归属域 | 生产者 | 消费者 |
|---|---|---|---|
| AgentRequest | Access Layer（入口） | AccessGateway / ProtocolAdapter | AgentOrchestrator |
| GoalContract | Agent Control Plane（目标） | AgentOrchestrator.extract_goal() | Planner, Reasoner, ContextOrchestrator |
| ContextPacket | Memory Layer（语义上下文） | ContextOrchestrator.assemble() | Planner, Reasoner, PromptComposer |
| Observation | Execution Layer（执行记录） | Tool/MCP/Worker/Retrieval 执行器 | DigestBuilder, Memory, ReflectionEngine |
| ObservationDigest | Cognition Layer（推理投影） | DigestBuilder / ToolManager | ContextOrchestrator, Memory, LLM |
| BeliefState | Cognition Layer（认知状态） | Reasoner / ReflectionEngine | ContextOrchestrator, Planner, Checkpoint |
| Checkpoint | Runtime Layer（恢复快照） | AgentOrchestrator / RecoveryManager | RecoveryManager, ReflectionEngine |
| AgentResult | Access Layer（出口） | ResponseBuilder | AgentFacade, AccessGateway |

**检查结论**：  
- 8 个对象分布在 5 个架构域中（Access 2 个、Control Plane 1 个、Memory 1 个、Execution 1 个、Cognition 2 个、Runtime 1 个）。  
- 无对象跨 Prompt 域（PromptComposer 是 ContextPacket 的消费者，不是生产者）。  
- 无对象跨 Recovery 域（RecoveryManager 是 Checkpoint 的消费者，不是生产者）。  
- 无对象跨 Worker 域（MultiAgentCoordinator 有独立契约，不复用主链路对象）。  
- ✅ 域归属唯一性通过。
