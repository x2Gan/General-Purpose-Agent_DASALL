## 0. Phase 0 研究学习证据链

### 本地证据清单

1. `docs/todos/contracts/WP-05-子域细化与ContractTestsTODO.md` 已将 T013-B 的代码目标与测试目标都固定为 `tests/contract/serialization/SerializationCompatibilityContractTest.cpp`，说明本轮必须聚焦“序列化兼容 contract tests”而非新增业务对象。
2. `contracts/include/agent/AgentRequest.h` 与 `contracts/include/agent/AgentRequestGuards.h` 已冻结 required 字段、时间字段与枚举边界；序列化兼容回归应优先覆盖这些高频消费字段。
3. `contracts/include/boundary/CompatibilityGuards.h` 已提供 `normalize_timeout_fields`，明确 `timeout_ms` 是规范字段，`timeout_seconds` 只能作为向后兼容输入。
4. `contracts/include/event/EventEnvelope.h` 与 `contracts/include/event/EventEnvelopeGuards.h` 已冻结 header 白名单与 payload 载体边界，适合作为“序列化字段漂移”负例入口。
5. `docs/todos/contracts/WP-05-子域细化与ContractTestsTODO.md` 的 WP05 完成标准要求覆盖“序列化稳定性、错误码/枚举兼容性、事件封套、ADR 边界字段”，因此 T013 要优先建立可复用的序列化兼容测试骨架。

### 外部参考清单

1. Protocol Buffers Language Guide, Updating A Message Type（https://protobuf.dev/programming-guides/proto3/#updating）：新增字段是 wire-safe，旧端应忽略不认识的新字段；可映射到本任务的“前向兼容忽略未知字段”断言。
2. JSON Schema Object Reference（https://json-schema.org/understanding-json-schema/reference/object）：`properties` 默认不会拒绝额外字段，只有显式关闭 additionalProperties 才会拒绝；可映射到本任务“对象级兼容默认允许扩展，边界白名单对象显式拒绝”。

### 对本任务的可落地启发

1. 序列化兼容测试需要同时覆盖两类对象：默认可扩展对象（如 AgentRequest）和显式白名单对象（如 EventEnvelope.header）。
2. 必须有正例覆盖 round-trip 稳定与 legacy 字段迁移，也必须有负例覆盖 required 字段缺失与白名单越权字段。
3. 兼容断言应尽量复用现有 guards，避免在测试中重新定义第二套业务规则。
4. 解析器实现应采用“最小可验证”策略：只解析本轮矩阵需要字段，其余字段显式忽略，以便后续 T014-T016 复用模式扩展。

## 2. Design 原子清单

1. D1：定义 T013 的最小序列化对象覆盖面。
- 输入依据：WP05 完成标准与现有 frozen guards。
- 产出路径：本文件第 3 节矩阵定义。

- 风险与回退：若对象范围膨胀到全仓库，回退为“高风险对象优先”。

2. D2：定义兼容测试的最小正负例集。
- 输入依据：Build 三件套与 contract test 二值判定要求。
- 产出路径：`tests/contract/serialization/SerializationCompatibilityContractTest.cpp`。
- 完成判定：至少 2 个正例 + 2 个负例，且每条断言能映射到 guard 结果。
- 风险与回退：若只剩 happy-path，补齐 required 缺失与白名单越权负例。

3. D3：锁定测试注册与验收路径。
- 输入依据：`tests/contract/CMakeLists.txt` 当前注册模式。
- 产出路径：`tests/contract/CMakeLists.txt`。
- 完成判定：`ctest --test-dir build-ci -N -R SerializationCompatibilityContractTest` 可发现 1 个测试。
- 风险与回退：若不可发现，先修注册，再看实现逻辑。

## 3. 序列化稳定性测试矩阵

| 序号 | 对象 | 兼容场景 | 断言类型 | 期望结果 |
|---|---|---|---|---|
| M1 | AgentRequest | V1 round-trip（必填字段 + timeout_ms） | 正例 | 反序列化后字段值保持稳定，`validate_agent_request_field_rules` 通过 |
| M2 | AgentRequest | 前向兼容：出现未知字段 | 正例 | 未知字段被忽略，已知字段可正常解析并通过 guard |
| M3 | AgentRequest | 向后兼容：legacy `timeout_seconds` 迁移到 `timeout_ms` | 正例 | 迁移后 `timeout_ms`/`deadline_at` 可用并通过 guard |

| M6 | EventEnvelope | header 出现非白名单字段 | 负例 | guard 失败，拒绝越权 header 字段 |
