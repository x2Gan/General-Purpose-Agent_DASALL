# WP02-T015 B2 枚举降级契约测试基线

最近更新时间：2026-03-14
关联阻塞项：B2
关联来源：WP02-T012 枚举规范、WP02-T014 评审纪要、WP02-T015 M2 冻结包

## 1. 任务理解

本基线只处理 B2：为“新增枚举值对旧消费方降级到 Unspecified”提供可执行、可留档的 contract tests 证据。

## 2. 测试约束

1. 依据 T012，新增枚举值时，旧消费方无法识别时必须走 `Unknown -> Unspecified` 降级路径。
2. 不通过删除旧值或重解释既有值来规避测试。
3. 测试关注兼容读取语义，不引入业务域私有枚举。

## 3. 基线范围

本基线覆盖以下最小场景：

1. 已知枚举值保持原值，不发生误降级。
2. 未知枚举值降级为 `Unspecified`。
3. `Unspecified` 本身必须是稳定兜底值。

## 4. 证据映射

| 证据项 | 落地点 |
|---|---|
| 兼容辅助头 | contracts/include/boundary/CompatibilityGuards.h |
| 契约测试文件 | tests/contract/smoke/CompatibilityContractTest.cpp |
| CTest 测试名 | dasall_contract_compatibility_test |

## 5. 通过标准

1. `dasall_contract_compatibility_test` 构建通过。
2. 测试用例证明 unknown 值稳定降级到 `Unspecified`。
3. 测试用例证明已知值不会被错误降级。

## 6. 风险与回退

1. 风险：实现侧把未知值映射到业务成功值。
   回退：统一回到 `Unspecified` 兜底路径，并补充失败日志。
2. 风险：不同模块各自定义未知值处理。
   回退：集中复用兼容辅助层，禁止分散解析。
3. 风险：后续新增枚举值未同步增加基线用例。
   回退：将本测试纳入 contract test 默认集合，缺失即阻断。
