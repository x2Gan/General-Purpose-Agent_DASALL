## 4. 冻结语义主结论

1. contracts 主对象采用 Stable/Blocked/Deferred 三层边界模型，防止流程中间态误入共享契约。
2. ContextPacket 保持语义上下文定位，不得承载 final_messages、provider_payload、rendered_prompt。
3. ReflectionDecision 与 RecoveryOutcome 保持建议权/执行权分层，禁止语义和调度职责串扰。
4. MultiAgentRequest、MultiAgentResult、WorkerTask 保持全局主控与协同子域分层，不得越权承载全局请求或最终输出语义。

## 6. 生效门禁（M1 Gate）

### 6.1 进入 WP-02 的准入条件

1. ADR-006/007/008 已对齐的核对单（T009/T010/T011）继续保持无阻塞冲突。
2. 评审纪要中的中低严重度问题已登记为显式遗留项并纳入后续任务跟踪。
3. 后续任务不得回退本冻结包中的主对象边界结论。

### 6.2 阻断条件（任一命中即阻断）

1. ContextPacket 出现 final_messages、provider_payload、rendered_prompt 任一字段。
2. ReflectionDecision 混入执行调度字段，或 RecoveryOutcome 回写失败归因语义。
3. MultiAgentRequest 复用 AgentRequest，或 MultiAgentResult 承担 AgentResult 语义。
4. WorkerTask 承载全局 Session/FSM 主态语义。

## 7. 显式遗留项（由评审意见转入）

### 7.1 R-001 非权威学习材料口径纠偏

1. 来源：T009/T012。
2. 问题：学习材料存在 ContextPacket 历史旧示例（包含 final_messages）。
3. 处置：转入文档治理任务，不影响 M1 生效。
4. 建议承接：后续文档治理工作包。

### 7.2 R-002 Orchestrator 简称语义歧义治理

1. 来源：T011/T012。
2. 问题：个别语境使用 Orchestrator 简称，可能被误读为协同层主控。
3. 处置：冻结期内统一使用 AgentOrchestrator 或 MultiAgentCoordinator 全称。
4. 建议承接：WP-04 设计评审与文档修订。

### 7.3 R-003 Deferred 对象复审机制

1. 来源：T007/T008/T012。
2. 问题：ToolRequest、ToolResult 处于 Deferred，需避免误判为永久禁止或提前外溢。
3. 处置：在 WP-05 启动时执行复审，决定升级或继续 Deferred。
4. 建议承接：WP-05 子域细化与 ContractTests。
