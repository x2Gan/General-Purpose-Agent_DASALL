## 2. 版本变更模板（V1）

### 2.1 模板字段

1. `change_id`：变更唯一标识。
2. `contract_domain`：变更所属 contracts 子域。
3. `previous_version`：变更前版本（SemVer）。
4. `target_version`：变更后版本（SemVer）。
5. `bump_kind`：`patch` / `minor` / `major`。
6. `compatibility`：`backward_compatible` / `partially_compatible` / `breaking`。
7. `summary`：变更摘要。
8. `rationale`：变更依据。
9. `migration_required`：是否要求迁移。
10. `breaking_review_required`：是否要求 breaking 专项评审。
11. `migration_plan`：迁移方案（breaking 必填）。
12. `deprecation_window`：弃用窗口（breaking 必填）。

### 2.2 判定规则

1. `major` 或 `compatibility=breaking` 视为 breaking 变更。
2. non-breaking 允许 `migration_required=false`，且不强制填写迁移方案。
3. breaking 必须满足：
- `breaking_review_required=true`
- `migration_required=true`
- `migration_plan` 非空
- `deprecation_window` 非空
4. `previous_version` 与 `target_version` 必须是三段式 SemVer（`X.Y.Z`）。
