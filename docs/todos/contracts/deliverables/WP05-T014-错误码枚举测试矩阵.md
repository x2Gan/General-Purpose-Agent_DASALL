### 外部参考清单

1. Protobuf Language Guide（https://protobuf.dev/programming-guides/proto3/#updating）：新增 enum 值属于兼容演进，旧端应维持可解析/可降级策略，避免直接破坏既有消费者。
2. Semantic Versioning 2.0.0（https://semver.org/）：公开 API 的不兼容变化属于 major 级变化；可映射为本任务中“错误码重编号/删掉哨兵”必须被测试识别为 breaking 迹象。

### 对本任务的可落地启发

1. 测试矩阵应同时覆盖“错误码编号快照稳定性”和“枚举生命周期兼容性”，避免只测单点。
2. 正例要覆盖：冻结编号不变、分段边界不漂移、未知枚举值降级、deprecated 值保留可读。
3. 负例要覆盖：区间外错误码被拒绝、缺失 `Unspecified` 哨兵被阻断。
4. 断言应尽量复用现有 guard/helper，保持与 frozen contracts 一致的语义来源。

1. D1：定义错误码兼容快照矩阵。
- 输入依据：`ResultCode.h` 冻结值与分段规则。

2. D2：定义枚举生命周期兼容矩阵。
- 输入依据：`EnumLifecycleGuards.h`。
- 产出路径：同上。
- 完成判定：至少包含未知值降级、deprecated 保留、缺失哨兵拒绝三类断言。
- 风险与回退：若缺失 breaking 负例，则补齐哨兵删除检测。

3. D3：锁定 Build 三件套与可发现性。
- 输入依据：`tests/contract/CMakeLists.txt` 注册模式。
- 产出路径：`tests/contract/CMakeLists.txt`。
- 完成判定：`ctest --test-dir build-ci -N -R ErrorCodeEnumCompatibilityContractTest` 构建后可发现 1 个测试。
- 风险与回退：若不可发现，先修注册，再看测试逻辑。
