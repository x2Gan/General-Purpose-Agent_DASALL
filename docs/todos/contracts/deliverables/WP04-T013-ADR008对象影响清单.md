### 1.3 可落地启发

1. T013 的受影响对象应锁定在 MultiAgentRequest、MultiAgentResult、WorkerTask、WorkerLease 四个协同子域对象，AgentRequest、AgentResult 和顶层 checkpoint 只作为上游锚点引用，不进入本任务的直接对象目录。
2. T013-B 应沿用 T001/T006 的聚合头模式，把 ADR-008 直接影响对象目录和既有 MultiAgentBoundaryGuards 汇总到单一 include，而不是提前实现 T014-T021 的实体对象。
3. 现有 `boundary/MultiAgentBoundaryGuards.h` 已冻结 request/result/task 三类禁区；T013 只需复用这些守卫，并在目录层面为 WorkerLease 预留位置，避免跨工作包实现租约守卫。
4. smoke gate 的职责是验证“对象目录 + 上游锚点分层 + 守卫可见性”，不是验证字段表或运行时调度算法。
5. 设计文档必须明确 AgentOrchestrator 仍是唯一最终输出方，防止 MultiAgentResult 或 WorkerTask 在后续任务中回退为 AgentResult / 全局状态承载体。

## 2. ADR-008 受影响对象全集

ADR-008 冻结的是“全局主控”和“协同子域编排”的边界。
T013 只输出直接受影响对象清单和聚合入口，不提前实现对象字段细节。

### 2.1 对象—锚点—职责归属表

| 对象 | 归属子域 | 上游锚点 | WP 编号 | 当前状态 | ADR-008 直接约束来源 |
|---|---|---|---|---|---|
| MultiAgentRequest | 协同请求 | AgentRequest | WP04-T014/T015 | Not Started | §5.1 |
| MultiAgentResult | 协同结果 | AgentResult | WP04-T016/T017 | Not Started | §5.2 |
| WorkerTask | 子任务执行单元 | MultiAgentRequest / 顶层任务图 | WP04-T018/T019 | Not Started | §5.3 |
| WorkerLease | 子任务租约元数据 | WorkerTask / 顶层 checkpoint 子域快照 | WP04-T020/T021 | Not Started | §5.3 / §5.4 |

**说明：**

1. AgentRequest、AgentResult 和顶层 checkpoint 已由 WP03 基线冻结，它们在 T013 中只作为 layering anchor 引用，不纳入 direct-impact catalog。
2. T013 不定义以上四个对象的字段表；字段冻结分别留给 T014-T021。
3. T013-B 只需要 compile-time aggregate entry，不新增运行时行为。

### 2.2 责任链（ADR-008 §4 固化）

```text
AgentRequest
  -> AgentOrchestrator decides single-agent or multi-agent mode
  -> MultiAgentRequest
  -> MultiAgentCoordinator creates WorkerTask graph and WorkerLease state
  -> Worker execution + ResultMerger
  -> MultiAgentResult
  -> AgentOrchestrator folds collaboration result into Observation / final AgentResult
```

**边界含义：**

1. MultiAgentRequest 是协同入口，不是第二个 AgentRequest。
2. MultiAgentResult 是协同回传值，不是最终 AgentResult。
3. WorkerTask / WorkerLease 是局部执行与租约对象，不得反向携带全局 Session/FSM/checkpoint 主控语义。

## 3. 直接影响对象与禁区映射

### 3.1 MultiAgentRequest 影响项（ADR-008 §5.1）

#### 3.1.1 语义边界

MultiAgentRequest 只表达协同子域请求，消费 AgentOrchestrator 已经裁定好的目标片段、计划片段和约束条件。

#### 3.1.2 最小输入槽位（仅影响清单，不是字段冻结）

| 输入槽位 | 作用 |
|---|---|
| `parent_request_id` | 关联顶层 AgentRequest |
| `parent_task_id` | 关联顶层任务或子图根节点 |
| `goal_fragment` / `subgoal` | 协同要处理的局部目标 |
| `plan_fragment` | 已裁定的局部计划片段 |
| `collaboration_mode` | 协同策略模式 |
| `worker_budget_guard` | Worker 级预算边界 |
| `permission_guard` | Worker 可执行权限边界 |
| `stop_conditions` | 局部退出条件 |

#### 3.1.3 禁区声明

1. 不得复用 `AgentRequest` 或 `agent_request_payload` 作为顶层套娃字段。
2. 不得重新承载全局 Session、最终用户交互或主状态机推进权。
3. 现有边界守卫由 `boundary/MultiAgentBoundaryGuards.h` 中的 `evaluate_multi_agent_request_field_boundary()` 复用。

### 3.2 MultiAgentResult 影响项（ADR-008 §5.2）

#### 3.2.1 语义边界

MultiAgentResult 只表达协同结果和后续建议，不承担最终回复提交权。

#### 3.2.2 最小结果槽位（仅影响清单，不是字段冻结）

| 结果槽位 | 作用 |
|---|---|
| `subtask_results` | 子任务结果集合 |
| `merged_result` | 合并后的协同产出 |
| `conflicts` | 冲突与分歧集合 |
| `worker_trace_refs` | Worker 轨迹引用 |
| `failure_summary` | 局部失败摘要 |
| `recommended_next_action` | 建议给 AgentOrchestrator 的下一步动作 |

#### 3.2.3 禁区声明

1. 不得引入 `agent_result` 或 `final_agent_response`，避免替代顶层 AgentResult。
2. 不得直接成为用户面最终输出。
3. 现有边界守卫由 `evaluate_multi_agent_result_field_boundary()` 复用。

### 3.3 WorkerTask 影响项（ADR-008 §5.3）

#### 3.3.1 语义边界

WorkerTask 只描述子任务执行单元，不承载全局主控态。

#### 3.3.2 最小执行槽位（仅影响清单，不是字段冻结）

| 执行槽位 | 作用 |
|---|---|
| `task_id` / `parent_task_id` | 子任务层级引用 |
| `lease_id` | 关联租约 |
| `worker_type` | Worker 能力选择 |
| `allowed_tools` | 当前子任务允许工具集 |
| `timeout` | 局部执行超时 |
| `idempotency_key` | 幂等与重放控制 |

#### 3.3.3 禁区声明

1. 不得携带 `global_session_state`、`global_fsm_state`、`session_fsm_state` 等顶层状态字段。
2. 不得反向成为 AgentOrchestrator 的主循环状态容器。
3. 现有边界守卫由 `evaluate_worker_task_field_boundary()` 复用。

### 3.4 WorkerLease 影响项（ADR-008 §5.3 / §5.4）

#### 3.4.1 语义边界

WorkerLease 只表达租约元数据和局部续约/释放语义，不承担结果对象或全局恢复入口职责。

#### 3.4.2 本任务冻结范围

1. T013 只在对象目录中登记 WorkerLease 是 ADR-008 直接影响对象。
2. T013 不实现 WorkerLease 对象或守卫；相关实现留给 T020/T021。
3. 顶层 checkpoint 与子任务租约快照必须分层，MultiAgentCoordinator 不能成为新的 resume 入口。
