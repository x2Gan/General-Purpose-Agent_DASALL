## 1. 任务范围

本交付仅完成“为每个核心术语补充一句话语义定义”，不包含：

1. 术语层级与主要消费者标注（WP01-T004）。
2. 顶层对象流图绘制（WP01-T005）。
3. 字段级命名冻结与字段约束细化（后续任务处理）。

## 2. 定义规则

1. 每个术语仅保留一句话定义，避免隐含实现细节。
2. 定义必须可被跨模块复用，不绑定具体类、库、序列化或存储实现。
3. 边界术语优先遵循 ADR-006/007/008 的职责裁定。
4. 同义别名不再重复定义，统一映射到 T002 的主名称。

## 3. 核心术语一句话定义

| 术语 | 一句话语义定义 |
|---|---|
| AgentRequest | AgentRequest 是系统接收一次用户或上游调用时的统一入口契约，承载执行所需的请求语义与基础元数据。 |
| AgentResult | AgentResult 是一次请求生命周期的统一输出契约，表达最终结果状态及可追溯的结果信息。 |
| GoalContract | GoalContract 是对任务目标、成功判据与约束边界的显式约定，用于约束后续决策和执行方向。 |
| ContextPacket | ContextPacket 是跨模块共享的语义上下文容器，用于承载推理所需的上下文信息而非最终消息载荷。 |
| Observation | Observation 是对工具、知识、人类或协同执行反馈的统一观测表达，用于驱动后续判断与状态更新。 |
| ObservationDigest | ObservationDigest 是面向后续推理消费的观测摘要表达，用于保留关键结论并降低原始观测噪声。 |
| BeliefState | BeliefState 是当前任务认知状态的结构化表达，用于区分已确认事实与待验证假设。 |
| ActionDecision | ActionDecision 是当前阶段可执行动作意图的语义决策表达，用于连接认知判断与执行路径选择。 |
| Checkpoint | Checkpoint 是用于恢复和续跑的最小状态快照契约，用于在中断后安全恢复任务推进。 |
| RuntimeBudget | RuntimeBudget 是对运行资源上限与消耗约束的统一表达，用于约束系统在预算内执行。 |
| ErrorInfo | ErrorInfo 是失败语义与失败上下文的统一表达，用于支持诊断、恢复判断与审计追踪。 |
| ResultCode | ResultCode 是跨模块共享的结果分类标识，用于稳定表达成功与失败类别而不依赖实现异常细节。 |
| PolicyDecision | PolicyDecision 是策略层对某动作是否允许或需附加条件的统一裁定表达。 |
| PromptPolicyDecision | PromptPolicyDecision 是针对消息装配结果的发送前治理裁定表达，用于约束可发送内容与策略修补结果。 |
| CompensationAction | CompensationAction 是对已发生副作用进行补偿处理的标准动作语义表达。 |
| ContextOrchestrator | ContextOrchestrator 是负责语义上下文组装与语义预算裁剪的职责主体，不负责最终消息渲染。 |
| PromptComposer | PromptComposer 是负责将语义上下文与提示资产装配为模型可消费请求的职责主体。 |
| PromptPolicy | PromptPolicy 是负责对装配后请求进行发送前合规与安全治理的职责主体。 |
| PromptComposeRequest | PromptComposeRequest 是驱动消息装配的输入契约，用于消费上下文和装配参数而不替代上下文定义。 |
| PromptComposeResult | PromptComposeResult 是消息装配产物及装配元数据的输出契约。 |
| PromptRelease | PromptRelease 是可被选择和发布的提示资产版本标识与内容载体。 |
| ContextAssembleRequest | ContextAssembleRequest 是触发上下文编排流程的输入契约，表达装配阶段与装配约束。 |
| ContextAssembleResult | ContextAssembleResult 是上下文编排流程的输出契约，表达最终上下文及装配元信息。 |
| CompressionRequest | CompressionRequest 是触发上下文压缩或裁剪行为的输入语义契约。 |
| CompressionResult | CompressionResult 是压缩或裁剪执行结果的输出语义契约，表达保留与剔除结果摘要。 |
| ReflectionEngine | ReflectionEngine 是负责失败归因与语义级改进行动建议生成的认知职责主体。 |
| ReflectionDecision | ReflectionDecision 是对失败后下一步方向的语义建议表达，不等同于运行时已执行控制动作。 |
| RecoveryManager | RecoveryManager 是负责恢复动作准入裁定与执行控制的运行时职责主体。 |
| RecoveryOutcome | RecoveryOutcome 是恢复执行结果与控制元数据的统一输出表达。 |
| RetryPolicy | RetryPolicy 是对可重试条件、次数边界与重试策略约束的统一语义表达。 |
| RecoveryPolicy | RecoveryPolicy 是对恢复路径选择与恢复准入规则的统一策略语义表达。 |
| CheckpointPolicy | CheckpointPolicy 是对检查点写入、更新与恢复触发规则的统一策略语义表达。 |
| CompensationPolicy | CompensationPolicy 是对补偿动作触发条件与补偿边界的统一策略语义表达。 |
| AgentOrchestrator | AgentOrchestrator 是拥有全局请求生命周期控制权与最终输出裁定权的主控职责主体。 |
| MultiAgentCoordinator | MultiAgentCoordinator 是在全局主控约束下负责多 Agent 子任务编排与结果汇聚的协同职责主体。 |
| MultiAgentRequest | MultiAgentRequest 是发起协同子域执行的输入契约，表达子目标、边界与协同约束。 |
| MultiAgentResult | MultiAgentResult 是协同子域执行结果的输出契约，表达协同产出与冲突摘要而非最终全局结果。 |
| WorkerTask | WorkerTask 是分派给执行单元的子任务执行契约，表达子任务边界与执行约束。 |
| WorkerLease | WorkerLease 是对子任务执行占用窗口与续约边界的租约语义表达。 |
| AgentRegistry | AgentRegistry 是对可用执行主体能力与可调度信息进行登记和查询的统一能力目录契约。 |
| ResultMerger | ResultMerger 是对多来源执行结果进行一致化合并并保留冲突信息的职责主体。 |
| SubTaskGraph | SubTaskGraph 是对子任务依赖关系与推进状态进行结构化表达的协同任务图契约。 |

## 4. 可追溯依据（代表性）

1. ContextPacket、PromptComposeRequest、PromptComposeResult 的分层边界来自 ADR-006 第 6 节。
2. ReflectionDecision 与 RecoveryOutcome 的分层边界来自 ADR-007 第 5 节。
3. MultiAgentRequest、MultiAgentResult、WorkerTask 的层级边界来自 ADR-008 第 5 节。
4. 语义优先于 schema 与顶层字段分层原则来自实施计划第 6.3 节与阶段 0 规则。
5. 主名称集合来自 WP01-T002 的去重核心术语清单。

## 5. 完成判定核对

对应 WP01-T003 完成判定“定义不依赖实现细节，能被跨模块复用”：

3. 边界相关术语定义已与 ADR 结论对齐。

## 6. 风险与回退策略

### 6.1 风险

1. 个别策略类术语在不同模块可能被理解为职责主体或配置对象，存在解释漂移风险。
2. SubTaskGraph 与 TaskGraph 语义邻近，后续任务中可能出现边界重叠讨论。
3. 若后续对象字段定义引入实现词汇，可能反向污染当前术语定义抽象层级。

### 6.2 回退策略

1. 若评审认为某定义过宽或过窄，按 ADR 和架构主链路语义做最小文本修订，不改主名称。
2. 若出现职责冲突，以 ADR 结论为准并在 T004 标注消费者分层。
3. 若后续字段设计与本定义冲突，优先调整字段命名或字段归属，不先改术语定义。
