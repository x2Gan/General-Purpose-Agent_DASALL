## 2. ADR-007 受影响对象全集

ADR-007 是 ReflectionEngine 与 RecoveryManager 职责边界的根裁定文档。
其约束直接影响以下三类 Recovery 子域对象。

### 2.1 对象—子系统—职责归属表

| 对象 | 归属子系统 | WP 编号 | 当前状态 | ADR-007 直接约束来源 |
|---|---|---|---|---|
| ReflectionDecision | cognition | WP04-T007/T008 | Not Started | §3.2、§5.1 |
| RecoveryRequest | runtime | WP04-T009/T010 | Not Started | §3.3、§5.2 |
| RecoveryOutcome | runtime | WP04-T011/T012 | Not Started | §3.3、§5.3 |

**说明：**
- Checkpoint 已在 WP03-T012/T013 冻结为恢复锚点对象，T006 只引用其边界，不重新定义。
- T006 不定义上述三个对象的字段表，只输出影响清单与聚合入口，后续由 T007-T012 逐项展开。

### 2.2 责任链（ADR-007 §4 固化）

```text
Tool/Workflow/MCP failure
  -> Observation + ErrorInfo
  -> Runtime persists Checkpoint and runtime counters
  -> ReflectionEngine -> ReflectionDecision
  -> RecoveryManager(ReflectionDecision, Checkpoint, Budget, Idempotency)
  -> RecoveryOutcome
```

**边界含义：**
1. ReflectionDecision 先输出失败语义解释，再由 RecoveryManager 判断是否允许真实执行恢复动作。
2. RecoveryRequest 是 RecoveryManager 的准入输入对象，不是第二个 ReflectionDecision。
3. RecoveryOutcome 是执行结果对象，不回写失败归因、不替代 ReflectionDecision。

## 3. ReflectionDecision 影响项（ADR-007 §5.1）

### 3.1 语义边界

ReflectionDecision 只表达失败语义判断和建议，不表达运行时调度细节。

### 3.2 允许承载的语义类别

| 语义类别 | 说明 |
|---|---|
| `decision_kind` | continue / retry_step / replan / abort_safe 等语义建议 |
| `rationale` | 对失败归因与建议原因的解释 |
| `confidence` | 建议可信度 |
| `relevant_observation_refs` | 关联的观测证据引用 |
| `plan_patch_hint` / `replan_hint` / `clarification_hint` / `skill_switch_hint` | 认知级提示，不是执行指令 |

### 3.3 禁止承载的运行时字段类别

| 禁止字段 | 禁止原因 | ADR-007 条款 |
|---|---|---|
| `retry_after_ms` | 调度时延属于 RecoveryManager 执行控制 | §5.1 |
| `backoff_strategy` | 退避策略属于运行时恢复控制 | §5.1 |
| `lease_extension` | 租约延长属于 runtime/协同控制 | §5.1 |
| `checkpoint_blob` | checkpoint 持久化属于 runtime | §5.1 |
| `circuit_breaker_transition` | 熔断迁移属于恢复准入控制 | §5.1 |

### 3.4 边界守卫映射

以上 5 个字段已由 boundary/RecoveryBoundaryGuards.h 中的：

1. `kReflectionSchedulingForbiddenFields`
2. `evaluate_reflection_decision_field_boundary()`

完成冻结，可由 T006-B 聚合入口直接复用。

## 4. RecoveryRequest 影响项（ADR-007 §5.2）

### 4.1 语义边界

RecoveryRequest 是 RecoveryManager 的恢复准入输入对象，承接反思建议与运行时约束，
但不替代 ReflectionDecision，也不提前表达 RecoveryOutcome 的执行结果。

### 4.2 最小输入槽位（仅影响清单，不是字段冻结）

| 输入槽位 | 作用 | 来源 |
|---|---|---|
| `reflection_decision` | 承接 ReflectionEngine 的语义建议 | ADR-007 §5.2 |
| `error_info` | 失败错误语义 | ADR-007 §5.2 |
| `latest_observation` | 最近执行观测 | ADR-007 §5.2 |
| `checkpoint` | 恢复锚点与待恢复状态 | ADR-007 §5.2 |
| `retry_counters` | 重试次数快照 | ADR-007 §5.2 |
| `runtime_budget_snapshot` | 预算与资源约束快照 | ADR-007 §5.2 |
| `idempotency_and_side_effect_report` | 是否允许真实重放的准入依据 | ADR-007 §3.3 / §5.2 |

### 4.3 禁区声明

RecoveryRequest 在 T006 阶段只冻结以下边界约束，具体字段命名留给 T009/T010：

1. 不得退化为第二个 ReflectionDecision，不承担失败归因长文本和认知补丁。
2. 不得预先写入 `executed_action`、`final_runtime_state`、`rejection_reason` 等 Outcome 结果层字段。
3. 必须显式消费 Checkpoint / Budget / Idempotency 约束，避免“只看反思建议就执行恢复”的单边设计。

---

## 5. RecoveryOutcome 影响项（ADR-007 §5.3）

### 5.1 语义边界

RecoveryOutcome 只表达 RecoveryManager 实际执行后的结果与控制元数据。

### 5.2 允许承载的结果类别

| 结果类别 | 说明 |
|---|---|
| `executed_action` | 实际采取的恢复动作 |
| `final_runtime_state` | 恢复后的运行态 |
| `updated_retry_counters` | 更新后的重试计数 |
| `checkpoint_ref` | 关联的 checkpoint 引用 |
| `compensation_result_ref` | 补偿动作结果引用 |
| `rejection_reason` / `escalation_reason` | 准入拒绝或升级原因 |

### 5.3 禁止承载的失败归因类别

| 禁止字段 | 禁止原因 | ADR-007 条款 |
|---|---|---|
| `failure_root_cause` | 失败归因属于 ReflectionDecision | §5.3 |
| `root_cause_analysis` | 根因分析属于认知层 | §5.3 |
| `belief_patch` | belief 修订不属于恢复结果对象 | §5.3 |
| `plan_patch_hint` | 计划语义修补属于 ReflectionDecision | §5.3 |

### 5.4 边界守卫映射

以上 4 个字段已由 boundary/RecoveryBoundaryGuards.h 中的：

1. `kRecoveryAttributionForbiddenFields`
2. `evaluate_recovery_outcome_field_boundary()`

完成冻结，可由 T006-B 聚合入口直接复用。
