最近更新时间：2026-03-13
任务状态：In Review
任务编号：WP01-T004
上游输入：WP01-T003 术语定义表 v1

## 1. 任务范围

本交付仅完成“标记每个术语所属层级和主要消费者”，不包含：

1. 顶层对象流图绘制（WP01-T005）。
2. 跨模块稳定对象标注图（WP01-T006）。
3. 禁止外溢对象清单（WP01-T007）。

## 2. 标注规则

1. 所属层级采用架构分层：Layer 7（apps）、Layer 6（runtime）、Layer 5（cognition/llm 支撑）、Layer 4（tools/memory/knowledge/multi_agent）、Cross-Layer（contracts）。
2. 术语类型分两类：
   - 跨模块稳定对象：属于 contracts 共享语义对象。
   - 模块内部术语：职责主体、策略组件或流程概念，不直接作为跨模块共享对象。
3. 主要消费者以“直接读写该术语语义的模块”为准，不展开到实现类级别。
4. 边界冲突时优先采用 ADR-006/007/008 结论。

## 4. 汇总判定

1. 跨模块稳定对象：28 项。

3. 已满足 T004 完成判定：可区分顶层共享对象与模块内部术语。

## 6. 风险与回退策略

### 6.1 风险

1. 个别术语同时具备“对象名”和“职责名”语义，评审时可能出现类型归类争议。
2. MultiAgentCoordinator 的“调度权在 Layer 6、实现域在 Layer 4”可能在评审记录中被简化丢失。
3. 策略类术语在后续任务中可能被对象化，导致类型分类变更。

### 6.2 回退策略

1. 若术语类型存在争议，先以“是否作为 contracts 共享对象”判定术语类型。
2. 若层级存在争议，先按 ADR 边界结论判定职责归属，再标注实现域。
3. 若后续对象化新增，保留本表为 v1，并在增量版本中显式记录分类变更。
