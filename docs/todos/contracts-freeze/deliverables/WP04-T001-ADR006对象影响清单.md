## 1. ADR-006 受影响对象全集

ADR-006 是 PromptComposer 与 ContextOrchestrator 职责边界裁定的根决策文档。
其约束直接影响以下三类对象，分属不同子系统。

### 1.1 对象—子系统—职责归属表

| 对象 | 归属子系统 | WP 编号 | 状态 | ADR-006 直接约束来源 |
|---|---|---|---|---|
| ContextPacket | memory | WP03-T010/T011 | **已冻结（M3 Baseline）** | §3.2、§6.1 |
| PromptComposeRequest | llm | WP04-T002/T003 | Not Started | §6.2 |
| PromptComposeResult | llm | WP04-T004/T005 | Not Started | §6.3 |

**说明：**
- ContextPacket 已在 WP03 冻结为 M3 基线对象，WP04 只在此基础上添加 Prompt 边界守卫，不修改其字段定义。
- PromptComposeRequest / PromptComposeResult 是 WP04 新增对象，字段定义在 T002/T004 中进行，T001 只定义边界约束。

### 1.2 调用顺序（ADR-006 §4 固化）

```
Runtime
  └─► ContextOrchestrator → ContextPacket
  └─► PromptRegistry → PromptRelease
  └─► PromptComposer(ContextPacket, PromptRelease) → PromptComposeResult
  └─► PromptPolicy → governed PromptComposeResult
  └─► LLMManager → model request
```

**边界含义：**
1. ContextPacket 先于 PromptComposeRequest 存在，后者消费前者。
2. PromptComposer 是 ContextPacket 的消费方，不是生产方。
3. PromptComposeResult 是 PromptComposer 的唯一产出，不回写 ContextPacket。

## 2. ContextPacket 禁区字段清单（ADR-006 §6.1/§8 新增至 Prompt 边界层面）

WP03-T010/T011 已定义 ContextPacket 的 10 类语义槽位和基本禁区字段。
ADR-006 §6.1/§8 在 Prompt 边界维度追加以下明确的消息层字段禁区：

### 2.1 消息渲染层字段（绝对禁止进入 ContextPacket）

| 禁止字段 | 禁止原因 | ADR-006 条款 |
|---|---|---|
| `final_messages` | 属于 PromptComposeResult 输出，不是语义上下文 | §6.1 禁令、§8 风控 |
| `provider_payload` | 模型厂商特定格式，属于 PromptComposeResult | §6.1 禁令 |
| `rendered_prompt` | 消息渲染产物，属于 PromptComposeResult | §6.1 禁令 |
| `prompt_bundle` | Prompt 装配包，属于 PromptComposeResult | §6.1 禁令 |
| `system_instructions` | Prompt 模板注入字段，属于 PromptComposer 职责 | §3.3 条款 |
| `few_shots` | Prompt 示例注入，属于 PromptComposer 职责 | §3.3 条款 |
| `output_schema` | 模型输出约束，属于 PromptComposer 职责 | §3.3 条款 |
| `tool_schemas` | 模型工具定义，属于 PromptComposer/PromptPolicy 职责 | §3.4 条款 |

### 2.2 边界守卫映射 → WP04-T001-B

以上 8 个字段须在 `PromptBoundaryContracts.h` 中记录为 `kContextPacketPromptForbiddenFields` 禁区数组，
并由 `evaluate_context_packet_prompt_field_boundary()` 函数实施守卫。

## 3. PromptComposeRequest 非上下文主控约束（ADR-006 §6.2）

### 3.1 职责边界

PromptComposeRequest 是装配请求对象（消费 ContextPacket），
**不是第二个上下文对象**，也不是 ContextPacket 的替代品。

### 3.2 允许字段（ADR-006 §6.2 明确）

| 字段 | 语义 |
|---|---|
| `stage` | 当前执行阶段 |
| `task_type` | 任务类型 |
| `context_packet` | 引用或嵌入 ContextPacket，消费方持有 |
| `prompt_release` | PromptRegistry 选出的 PromptRelease 引用 |
| `visible_tools` | 本轮可见工具标识集合 |
| `model_route` | 目标模型路由 |
| `output_schema_ref` / `response_format` | 输出结构约束引用 |

### 3.3 禁止字段（ADR-006 §6.2 对应的边界禁区）

PromptComposeRequest **不允许**包含以下类型的字段，否则它将演化为第二个上下文主控对象：

| 禁止字段/类型 | 禁止原因 | ADR-006 条款 |
|---|---|---|
| `memory_snapshot` 类字段 | Working Memory 的原始内容属于 ContextOrchestrator 范畴 | §3.2/§7 方案 B 不采纳 |
| `retrieval_candidates` 类字段 | 候选片段检索属于 ContextOrchestrator 职责 | §3.2/§3.3 |
| `context_packet_internal_*` 类字段 | ContextPacket 的内部数据不应透传给装配请求 | §6.1 |
| `knowledge_fragments` 类字段 | 未经 ContextOrchestrator 处理的知识片段 | §7 方案 B 不采纳 |

### 3.4 边界守卫映射 → WP04-T001-B

以上字段须在 `PromptBoundaryContracts.h` 中记录为 `kComposeRequestContextOwnershipForbiddenFields` 数组，
并由 `evaluate_compose_request_field_boundary()` 函数实施守卫。

## 4. PromptComposeResult 无 memory 写回约束（ADR-006 §6.3/§3.3）

### 4.1 职责边界

PromptComposeResult 只表达**装配产物与装配元数据**，
**不反向承担 memory 写回、候选信息检索或知识召回语义**（ADR-006 §3.3 明文禁止）。

### 4.2 允许字段（ADR-006 §6.3 明确）

| 字段 | 语义 |
|---|---|
| `messages` | provider-neutral prompt payload |
| `selected_prompt_id` | 选用的 PromptSpec 标识 |
| `selected_version` | 选用的版本 |
| `pruned_sections` | 装配过程中裁剪的内容记录 |
| `estimated_tokens` | 预估的 token 数量 |
| `composition_warnings` | 装配异常信号（如超预算警告）|

### 4.3 禁止字段（ADR-006 §3.3 边界禁区）

| 禁止字段/类型 | 禁止原因 | ADR-006 条款 |
|---|---|---|
| `memory_write_back` 类字段 | PromptComposer 不得写回 memory | §3.3 条款 5 |
| `context_update` 类字段 | ContextPacket 更新由 ContextOrchestrator 负责 | §3.2 |
| `belief_patch` 类字段 | BeliefState 修订由 Cognition/memory 负责 | §3.3 条款 3/§7 |
| `knowledge_recall` 类字段 | 知识召回属于 ContextOrchestrator 范畴 | §3.3 条款 1 |
| `history_update` 类字段 | 历史记录写回属于 memory 子系统 | §3.3 条款 3 |

### 4.4 边界守卫映射 → WP04-T001-B

以上字段须在 `PromptBoundaryContracts.h` 中记录为 `kComposeResultMemoryWritebackForbiddenFields` 数组，
并由 `evaluate_compose_result_field_boundary()` 函数实施守卫。

## 5. 对象间关系约束图

```
┌──────────────────────────────────────────────────────────────────────────┐
│ ContextOrchestrator [memory subsystem]                                   │
│   产出 ContextPacket（10 类语义槽位，禁消息渲染字段）                          │
└─────────────────────────────┬────────────────────────────────────────────┘
                              │ 单向消费（ADR-006 §4、§3.3）
                              ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ PromptComposer [llm subsystem]                                           │
│   输入：PromptComposeRequest（含 ContextPacket 引用 + PromptRelease）        │
│   产出：PromptComposeResult（消息层 + 装配元数据，禁回写 memory）               │
└─────────────────────────────┬────────────────────────────────────────────┘
                              │ 单向下传（ADR-006 §4）
                              ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ PromptPolicy → LLMManager                                                │
│   治理与模型适配（ADR-006 §3.4，不回头替代 ContextOrchestrator）               │
└──────────────────────────────────────────────────────────────────────────┘
```

**ADR-006 三大禁区总结：**
1. **ContextPacket 禁区**：不得含消息渲染层字段（final_messages / rendered_prompt / provider_payload 等）。
2. **PromptComposeRequest 禁区**：不得成为第二上下文主控（memory_snapshot / retrieval_candidates 等）。
3. **PromptComposeResult 禁区**：不得回写 memory（memory_write_back / belief_patch / context_update 等）。
