# WP05-T008 任务子域对象定义

最近更新时间：2026-03-20
任务状态：Done
任务编号：WP05-T008
上游输入：WP-04 多 Agent 边界冻结包、ADR-008、DASSALL Agent 架构文档 4.8/6.11、DASALL contracts 冻结实施计划阶段 8

## 0. Phase 0 研究学习证据链

### 本地证据清单

1. ADR-008 明确 MultiAgentCoordinator 仅拥有协同子任务图的局部编排权，不得回收顶层 AgentOrchestrator 的 Session/FSM 主控。
2. WP04 已冻结 `WorkerTask` 与 `WorkerLease`，但 task 子域仍缺少统一“对象目录 + 越权字段门禁”入口。
3. 架构文档 4.8/6.11 明确子任务图（SubTaskGraph）是协同子域中的中间契约对象，不是顶层会话状态机。
4. WP05-T001 rollout 结论要求 Wave3（task + event）继续执行“子域对象细化 + 越权阻断”纪律。

### 外部参考清单

1. Martin Fowler《Consumer-Driven Contracts》：建议 provider 维持最小且可演进的契约面，避免把消费者未要求的内部控制语义扩散到共享契约。
2. Protobuf Language Guide（editions）：强调 schema 演进应优先“新增字段而非破坏性复用”，并通过兼容约束维持可前后向演进。

### 对本任务的可落地启发

1. task 子域需要统一目录对象，明确“哪些对象属于 task contracts，哪些只是上游锚点”。
2. 对于可扩展但尚未冻结完整字段的对象（如 SubTaskGraph），先给出最小稳定对象骨架，避免提前绑定 runtime 策略算法。
3. 越权阻断必须同时覆盖：顶层 Session/FSM、checkpoint/resume 入口、最终结果语义。
4. contract tests 必须包含正例与负例，并验证越权拒绝 reason 的稳定性。

## 1. 任务理解

本任务只处理 WP05-T008：

1. 新增 task 子域聚合对象入口 `TaskDomainContracts.h`。
2. 新增 task 子域越权守卫 `TaskDomainGuards.h`。
3. 新增并注册 `TaskDomainContractTest.cpp`，验证目录对象与越权阻断。

本任务不处理：

1. runtime 中的 AgentRegistry 匹配算法、ResultMerger 合并算法、租约调度策略。
2. 对 WP04 已冻结对象（WorkerTask/WorkerLease）进行字段回改。
3. event/llm 等后续子域任务内容。

## 2. 约束与边界

### 2.1 task 子域允许承载

1. 协同子域执行对象：WorkerTask。
2. 协同子域租约对象：WorkerLease。
3. 协同子域任务图快照对象：SubTaskGraph。

### 2.2 task 子域禁止承载

1. 顶层 Session/FSM 主控字段：`session_id`、`global_session_state`、`global_fsm_state`、`session_fsm_state`。
2. 顶层恢复入口字段：`checkpoint_ref`、`resume_token`。
3. 顶层最终结果字段：`agent_result`、`final_agent_response`、`merged_result`。

## 3. Design 原子清单

1. D1：冻结 task 子域对象目录与上游锚点。
- 输入依据：ADR-008、WP04 冻结包。
- 产出路径：`contracts/include/task/TaskDomainContracts.h`。
- 完成判定：可枚举 WorkerTask/WorkerLease/SubTaskGraph，且可查询对象上游锚点。
- 风险与回退：若目录吸纳顶层对象，回退到“仅 task 子域对象”最小集合。

2. D2：冻结 SubTaskGraph 最小对象骨架。
- 输入依据：架构文档 4.8/6.11。
- 产出路径：`contracts/include/task/TaskDomainContracts.h`。
- 完成判定：SubTaskGraph 至少具备图标识、根任务锚点、任务节点集合；不承载顶层主控字段。
- 风险与回退：若字段扩张到调度算法参数，回退为最小快照结构。

3. D3：新增 task 子域越权守卫与测试入口。
- 输入依据：WP05-T008 TODO 三件套要求。
- 产出路径：`contracts/include/task/TaskDomainGuards.h`、`tests/contract/task/TaskDomainContractTest.cpp`。
- 完成判定：至少 1 个正例 + 1 个负例；越权 reason 可稳定断言。
- 风险与回退：若 only-happy-path，则补充 forbidden field 分类断言。
