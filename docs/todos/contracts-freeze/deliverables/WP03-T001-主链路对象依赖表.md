# WP03-T001 主链路对象依赖表

最近更新时间：2026-03-15

上游输入：WP01-T013 M1 冻结包、WP02-T015 M2 冻结包

## 1. 任务理解

本任务只处理 WP03-T001：明确主链路对象全集与依赖顺序，交付“主链路对象依赖表”。

本任务不展开字段设计、不定义对象内部结构、不进入 WP03-T002 及后续任务。

## 2. 约束与边界

### 2.1 直接约束（可追溯）

1. 来源 plans：阶段 2 要先冻结单 Agent 主链路最小闭环对象，优先对象包含 AgentRequest、AgentResult、GoalContract、Observation、ObservationDigest、BeliefState、ContextPacket、Checkpoint。
2. 来源总 TODO：M3 目标是 AgentRequest 到 AgentResult 对象链完整闭环，且 WP-03 以前置 M2 为准入条件。
3. 来源 WP-03 TODO：T001 完成判定要求明确 AgentRequest、GoalContract、ContextPacket、Observation、ObservationDigest、BeliefState、Checkpoint、AgentResult 顺序。
4. 来源 M1 冻结包：上述对象均已进入 Stable 冻结对象集合，可作为主链路候选全集。
5. 来源 ADR-006：ContextPacket 只能承载语义上下文，不得承载消息渲染字段。
6. 来源 ADR-007：retry_step 属于语义建议，恢复动作执行权在 RecoveryManager；因此 Checkpoint 处于主链路中的“恢复入口前态”。

### 2.2 边界与非目标

边界：

1. 只冻结对象级依赖顺序，不冻结字段与枚举。
2. 只覆盖单 Agent 最小闭环，不引入 MultiAgentRequest、WorkerTask、RecoveryOutcome 等 WP-04/WP-05 对象。
3. 不改写 ADR 已冻结职责边界。

非目标：

1. 不定义 AgentRequest/GoalContract/ContextPacket 字段表。
2. 不定义 Observation 来源分类细则。
3. 不定义 Checkpoint 恢复字段。

## 3. 方案对比与决策

### 3.1 方案 A：线性最小链

顺序：
AgentRequest -> GoalContract -> ContextPacket -> Observation -> ObservationDigest -> BeliefState -> Checkpoint -> AgentResult

优点：

1. 顺序简单，评审成本最低。
2. 与 T001 完成判定中的对象全集完全一致。

缺点：

1. 无法表达“Checkpoint 对 Observation 与 BeliefState 的双依赖”。
2. 容易被误解为 Checkpoint 严格晚于 AgentResult 前最后一步，而不是循环中的可持久化节点。

### 3.2 方案 B：分层依赖链（推荐）

主干顺序：
AgentRequest -> GoalContract -> ContextPacket -> Observation -> ObservationDigest -> BeliefState -> Checkpoint -> AgentResult

依赖补充：

1. ObservationDigest 依赖 Observation（推理消费层）。
2. BeliefState 依赖 ObservationDigest（认知事实层）。
3. Checkpoint 依赖 Observation 与 BeliefState（恢复最小状态层）。
4. AgentResult 依赖 ContextPacket 与本轮有效 ObservationDigest/BeliefState 快照（结果汇聚层）。

优点：

1. 保留 T001 要求的线性顺序，同时显式表达关键非线性依赖。
2. 与 ADR-006/007 的边界更一致，避免 ContextPacket 与恢复语义串扰。

缺点：

1. 评审时需要同时检查主干顺序与依赖补充两部分。

### 3.3 决策

采用方案 B。

取舍理由：在不跨出 T001 范围的前提下，方案 B 能把“顺序”与“依赖”同时冻结，减少后续 T006/T008/T009/T012 返工风险。

## 6. 风险与回退

1. 风险：把顺序误解为严格单向流水，忽略 Checkpoint 的恢复入口语义。
回退：在后续 T012 明确 Checkpoint 为“可重入恢复点”，并保留本表中的双依赖说明。
2. 风险：ObservationDigest 与 Observation 边界被重新混合，导致 cognition/memory/audit 耦合。
回退：以计划阶段 2 和 WP03-T008 约束为准，若发生混合则退回本表与 T008 边界说明重审。
3. 风险：已解阻后忽略 M2 Gate，导致 WP-03 对象再次发明横切语义。
回退：在 T002 起每个对象评审中强制复核 WP02-T013 Checklist 与 M2 Gate 条件。
