## 2. Phase 0 研究学习结果

### 2.1 本地证据清单

1. 架构主链路核心对象冻结要求：AgentRequest、GoalContract、Observation、BeliefState、Checkpoint、AgentResult，且主流程与异常流程使用同一套契约。
来源：docs/architecture/DASSALL_Agent_architecture.md（3.8 核心契约层）。
2. 单 Agent 运行步骤中明确了主链路编排顺序：AgentRequest -> ContextPacket -> Planner/Reasoner -> Observation Digest -> 输出；并强调 GoalContract、BeliefState、Checkpoint 在循环中显式维护。
来源：docs/architecture/DASSALL_Agent_architecture.md（6.2 详细运行步骤）。
3. contracts 冻结计划阶段 2 明确 WP03 主链路优先对象集合：AgentRequest、AgentResult、GoalContract、Observation、ObservationDigest、BeliefState、ContextPacket、Checkpoint。
来源：docs/plans/DASALL_contracts冻结实施计划.md（阶段 2）。
4. ADR-006 规定 ContextPacket 只能承载语义上下文，不得携带 final_messages、provider_payload、rendered_prompt。
来源：docs/adr/ADR-006-context-orchestrator-vs-prompt-composer.md（6.1）。
5. ADR-007 规定 retry/replan 属于认知建议，恢复执行裁定归 RecoveryManager，说明 Checkpoint 是主控恢复入口语义而非推理对象。
来源：docs/adr/ADR-007-reflection-engine-vs-recovery-manager.md（7.2）。
6. ADR-008 规定 AgentOrchestrator 拥有全局主控与最终输出权，MultiAgentCoordinator 只拥有协同子域局部编排权，避免把多 Agent 对象混入 WP03-T001 单 Agent 主链路。
来源：docs/adr/ADR-008-agent-orchestrator-vs-multi-agent-coordinator.md（3.4、4）。

### 2.2 外部参考清单

1. Microsoft Azure API design best practices：推荐 contract-first、资源模型不暴露内部实现、新增字段优先于破坏性重命名/删除，适用于主链路对象演进策略。
链接：https://learn.microsoft.com/en-us/azure/architecture/best-practices/api-design
2. Protobuf Best Practices：不要复用字段编号、避免改字段类型、枚举包含 Unspecified 缺省值、避免一次性超大消息体，适用于 contracts 的向后兼容与对象拆分。
链接：https://protobuf.dev/best-practices/dos-donts/
3. Consumer-Driven Contracts（Martin Fowler）：消费者只校验“刚好够用”的字段集合，用契约测试守住消费者依赖面，降低 provider 演进时的隐式耦合破坏。
链接：https://martinfowler.com/articles/consumerDrivenContracts.html

### 2.3 可落地启发（仅针对 WP03-T001）

1. 主链路对象顺序必须稳定，但对象内部字段应延后到各自子任务冻结，避免 T001 越界。
2. MainFlowContracts 聚合入口应只做“稳定对象编排与顺序约束”，不承载字段语义。
3. 需要提供可程序化的顺序守卫（例如 direct successor 判断）以支持 contract smoke 测试。
4. 负例测试要验证“错序对象”被拒绝，符合 consumer-driven contract 的边界验证思想。
5. 保持 ContextPacket 与 Checkpoint 边界分离，避免消息装配或恢复执行语义侵入主链路对象定义。

回退：在 MainFlowContracts 内以占位入口类型承载，待 T009 冻结后替换为正式对象。
3. 风险：测试未覆盖负例导致顺序回归难发现。
回退：保持正负例双分支 smoke 测试，并纳入 dasall_contract_tests。
