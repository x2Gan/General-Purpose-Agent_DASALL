## 0. Phase 0 研究学习证据链

### 本地证据清单

1. 架构文档 5.3.5 明确 memory 需要支持写回闭环，沉淀可复用经验，而不是只保存会话文本。
2. 架构文档 5.3.6 要求 Session Store 支持事实和经验级对象，避免把 runtime 控制字段混入共享对象。
3. ADR-006 指出 ContextPacket 负责上下文装配，不应由 memory 持久对象反向承载装配状态。
4. ADR-007 指出 retry、checkpoint、recovery 归属 runtime，memory 对象不能承担恢复执行语义。
5. WP05-T006 已冻结 Turn/Session/SummaryMemory，T007 需补齐事实与经验沉淀面，并保持与主链路对象分层。

### 外部参考清单

1. Protobuf Schema Evolution 实践：稳定契约应维持最小必填字段并使用可选扩展字段，避免破坏性字段语义漂移。
2. Consumer-Driven Contracts 实践：共享契约只暴露消费方所需最小面，避免把 provider 私有字段扩散到系统主链路。

### 对本任务的可落地启发

1. MemoryFact 应建模为可追溯的原子事实对象，要求来源引用和置信度，避免不可审计写回。
2. ExperienceMemory 应建模为可复用的经验条目，强调触发条件与推荐动作，不承载调度控制。
3. 两个对象都应通过字段边界守卫阻断 SessionContext、runtime 控制字段、provider 私有字段污染。
4. contract test 需覆盖正例与负例，并显式验证越权字段分类拒绝。

## 2. 约束与边界

### 2.1 MemoryFact 边界

MemoryFact 允许承载：

1. 稳定标识与归属：fact_id、session_id。
2. 事实内容与追溯：fact_text、source_turn_ids、source_observation_refs。
3. 质量与生命周期：confidence_score、created_at、valid_until、superseded_by_fact_id。
4. 审计信息：evidence_digest、tags。

MemoryFact 明确禁止承载：

1. SessionContext/ContextPacket 装配字段：recent_history、current_goal_summary、policy_digest、belief_state_summary。
2. runtime 控制字段：fsm_state、retry_count、pending_action、scheduler_slot。
3. provider 私有载荷字段：provider_payload、rendered_prompt、raw_embedding、token_logprobs。

### 2.2 ExperienceMemory 边界

ExperienceMemory 允许承载：

1. 稳定标识与归属：experience_id、session_id。
2. 经验语义：lesson_summary、trigger_condition、recommended_action。
3. 追溯与适用性：source_fact_ids、source_turn_ids、applicable_domains、risk_notes。
4. 生命周期与质量：created_at、expires_at、effectiveness_score、superseded_by_experience_id、tags。

ExperienceMemory 明确禁止承载：

1. runtime 控制字段：fsm_state、retry_after_ms、recovery_action、scheduler_slot。
2. prompt/provider 字段：rendered_prompt、provider_payload、model_route、temperature。
3. checkpoint 字段：checkpoint_id、pending_action、working_memory_snapshot、retry_count。
