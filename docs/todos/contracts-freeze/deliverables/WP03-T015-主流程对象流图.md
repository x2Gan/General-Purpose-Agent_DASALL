### 2.3 对本任务的可落地启发

1. **关联字段是流图的"管线"**：request_id 贯穿 AgentRequest → GoalContract → ContextPacket → BeliefState → AgentResult；observation_id 贯穿 Observation → ObservationDigest；goal_id 贯穿 GoalContract → AgentResult。端到端测试必须验证这些管线不断裂。
2. **每段流转都必须通过对应 Guard**：不能只验证对象构造成功，还必须验证每个对象都能通过其冻结的 Guard（L1 必填 + L2 边界）。
3. **负例应覆盖关联断裂**：端到端测试的负例不应仅是字段缺失，更应是跨对象关联不一致（如 GoalContract.request_id ≠ AgentRequest.request_id）。
4. **MainFlowContracts.kCanonicalOrder 是流图的可执行编码**：测试应遍历 kCanonicalOrder 验证 8 节点完整性和顺序。
5. **流图文档应可追溯到代码**：每个节点标注对应 .h 文件和 Guard 函数名，每条边标注关联字段。

### 3.5 端到端流转示意（单轮 Tool 调用场景）

```
1. 用户输入 → AgentRequest (request_id="req-001", user_input="查询天气")
       ↓ [request_id=req-001]
2. 目标提取 → GoalContract (goal_id="goal-001", request_id="req-001",
                            goal_description="查询北京天气",
                            success_criteria="返回温度和天气状况")
       ↓ [request_id=req-001, current_goal_summary←goal_description]
3. 上下文组装 → ContextPacket (request_id="req-001",
                               user_turn="查询天气",
                               current_goal_summary="查询北京天气")
       ↓
4. Reasoning → ToolCall decision
       ↓
5. 工具执行 → Observation (observation_id="obs-001", source=ToolExecution,
                           success=true, payload='{"temp":22,"desc":"晴"}')
       ↓ [observation_id=obs-001]
6. 摘要构建 → ObservationDigest (observation_id="obs-001",
                                 summary="北京当前22°C，晴天",
                                 confidence=0.95)
       ↓ [request_id=req-001]
7. 认知更新 → BeliefState (request_id="req-001",
                           confirmed_facts=["北京22°C，晴天"],
                           confidence=0.95)
       ↓
8. 检查点 → Checkpoint (checkpoint_id="ckpt-001", state=Succeeded,
                        step_id="tool-weather-001")
       ↓ [checkpoint_ref=ckpt-001, request_id=req-001, goal_id=goal-001]
9. 结果构建 → AgentResult (result_id="res-001", status=Completed,
                           result_code=1001, response_text="北京22°C，晴",
                           task_completed=true, request_id="req-001",
                           goal_id="goal-001", checkpoint_ref="ckpt-001")
```

## 6. 风险与回退

| 风险 | 影响 | 回退方案 |
|---|---|---|
| 部分对象仍为 Tag（AgentRequest/GoalContract/Observation/ObservationDigest/BeliefState 在 MainFlowContracts 中仍是 Tag 别名） | E2E 测试无法直接通过 MainFlowContracts Entry 类型构造完整对象 | 测试直接 include 各 .h 构造实际 struct，不依赖 Entry 类型别名 |
| E2E 测试可能与各单体测试重复 | 测试维护成本增加 | E2E 测试专注关联字段一致性和全链路通过，不重复单字段负例 |
