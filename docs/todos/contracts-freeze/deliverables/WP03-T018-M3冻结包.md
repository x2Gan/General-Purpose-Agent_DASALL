## 5. 冻结语义主结论

### 5.1 主链路 8 对象流转闭环

AgentRequest → GoalContract → ContextPacket → Observation → ObservationDigest → BeliefState → Checkpoint → AgentResult

- 每个节点有明确的必填/可选字段定义与 Guard 守卫。
- 相邻节点通过 request_id/goal_id/observation_id 等关联字段串联。
- 端到端流转可编译、可测试闭环（T015-B 9/9 sub-tests passed）。

### 5.2 五域禁止边界

| 禁止域 | 说明 | 覆盖 |
|---|---|---|
| Prompt | 消息渲染内容（final_messages, rendered_prompt, provider_payload） | 8 对象全覆盖 |
| Recovery | 恢复动作执行（failure_root_cause, retry_after_ms, backoff_strategy） | 8 对象全覆盖 |
| Worker | Worker 协同细节（worker_results, merged_result, worker_task_id） | 8 对象全覆盖 |
| Runtime | 运行时私有状态（thread_handle, socket_fd, event_loop_ref） | 8 对象全覆盖 |
| Provider | 提供者私有状态（api_key, model_endpoint, token_count） | 8 对象全覆盖 |

### 5.3 ADR 一致性

| ADR | 高扇出对象 | 结论 |
|---|---|---|
| ADR-006 | ContextPacket | ContextPacket ≠ Prompt 域，10 槽位覆盖完整 — **Go** |
| ADR-007 | Checkpoint | Checkpoint ≠ Recovery 域，状态限于运行进度 — **Go** |
| ADR-008 | AgentResult | AgentResult ≠ Worker 域，不暴露协同细节 — **Go** |

### 5.4 域归属分布

| 域 | 对象 |
|---|---|
| AccessLayer | AgentRequest, AgentResult |
| ControlPlane | GoalContract |
| MemoryLayer | ContextPacket |
| ExecutionLayer | Observation |
| CognitionLayer | ObservationDigest, BeliefState |
| RuntimeLayer | Checkpoint |

## 7. 生效门禁（M3 Gate）

### 7.1 进入 WP-04 的准入条件

1. WP-03 CI 门禁脚本（wp03_contract_gate.sh）通过，37/37 contract tests passed。
2. M3 Checklist 10 Gate 全部 Pass（M3ChecklistGuards.h 程序化验证）。
3. 后续任务不得回退本冻结包中 8 对象的语义边界与禁止域结论。

### 7.2 阻断条件（任一命中即阻断）

1. 任何对象出现 5 域禁止字段中任一字段。
2. 端到端关联边断裂（request_id/goal_id 等关联字段不一致）。
3. ADR-006/007/008 一致性结论被改变但未经评审。
4. CI 门禁脚本返回非零状态码。

### 7.3 变更管理规则

1. 新增必填字段：必须先扩展对应 Guard（L1 层），补充测试，再更新冻结包。
2. 新增可选字段：只需扩展 L2/L3 层 Guard，补充测试。
3. 变更禁止字段：必须先修订对应 ADR，经评审通过后更新所有受影响 Guard。
4. 变更域归属：必须经全体评审，更新 T016 互斥矩阵和 T015 流转图。
