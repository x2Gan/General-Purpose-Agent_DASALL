# WP02-T014 评审纪要

最近更新时间：2026-03-16
任务状态：Done
任务编号：WP02-T014
评审范围：WP02-T001 至 WP02-T013 交付物

## 1. 任务理解

本任务只处理 WP02-T014：组织横切基础对象评审，输出可追溯评审纪要。

本任务完成判定：主要争议项形成“决议”或“遗留项”。

本任务不发布冻结包（T015）且不改动代码实现。

## 2. 约束与边界

### 2.1 可追溯约束

1. 输入依据限定为 T001-T013 交付物。
2. 评审依据以 T013 Checklist v1 为执行模板。
3. 不改写 ADR-006/007/008 结论；若冲突只能形成遗留并升级。
4. 评审输出需覆盖兼容性、字段演进、错误语义、预算、标识、时间、事件封套、枚举八类横切主题。

### 2.2 边界

1. 本任务只产出评审纪要，不新增契约规则。
2. 本任务只对争议点给出决议或遗留，不直接落地实现。
3. 本任务不替代 T015 冻结发布动作。

### 2.3 非目标

1. 不做代码级 merge/revert。
2. 不做版本号升级发布。
3. 不新增业务域私有字段。

## 3. 方案对比与决策

### 3.1 方案 A：仅记录结论摘要

优点：

1. 文档短，产出快。

缺点：

1. 缺失争议-证据-决议链路，难追溯。
2. 无法指导 T015 的冻结门禁。

### 3.2 方案 B：结构化评审纪要（争议项清单）

优点：

1. 每个争议项都有来源、讨论、决议/遗留和后续动作。
2. 可直接作为 T015 的输入与审计依据。

缺点：

1. 纪要维护成本更高。

### 3.3 决策

采用方案 B。

取舍理由：满足“主要争议项形成决议或遗留项”的完成判定，并可直接服务后续冻结发布。

| 编号 | 争议主题 | 来源任务 | 讨论要点 | 结论 |

| D1 | tool 与 provider 边界 | T004 | 远程失败归类易混淆 | 决议：远程协议/可用性问题优先归 provider，本地执行链路归 tool |

| D4 | BudgetSnapshot 计算口径 | T008 | remaining 是否允许手工赋值 | 决议：remaining 统一按 max-current 推导，禁止手工不一致写入 |
| D5 | 标识透传与父子关系 | T009 | trace_id 是否允许子链路重写 | 决议：request/session/trace 入口生成并透传；task 用 parent_task_id 建层次 |
